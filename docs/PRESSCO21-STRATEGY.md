## 축 1: 판매 촉진 (즉시 실행) — 노션 CMS + Next.js

### Idea 2: 튜토리얼/레시피 허브 ---- 임팩트 5/5

단계별 압화 만들기 가이드 + 재료 → 메이크샵 "바로 구매" 연결

- **노션 DB**: 튜토리얼(제목, 난이도, 소요시간) + 재료목록(재료명, 수량, 가격, 메이크샵 URL) + 단계(순서, 사진, 설명)
- **페이지**: `/tutorials` 카드 그리드, `/tutorials/[slug]` 단계별 상세 + 재료 사이드바
- 기존 YouTube 영상을 텍스트+사진으로 재가공 → 초기 콘텐츠 확보

### Idea 12: 재료 조합 가이드 ---- 임팩트 5/5

"이 꽃 + 이 도구 = 이 작품" 조합으로 세트 구매 유도

- **노션 DB**: 조합(완성사진, 난이도, 계절, 스타일) + 조합재료(재료명, 가격, 상품URL) → rollup 총비용 자동합산
- **페이지**: `/combos` 갤러리, `/combos/[id]` 상세 + "한 번에 구매" 버튼
- 초보 고객의 "뭘 사야 하지?" 의사결정 도움 → 객단가 상승

### Idea 4: 시즌 캠페인 랜딩 ---- 임팩트 5/5

봄/여름/가을/겨울 프로모션 페이지를 노션에서 빠르게 제작

- **노션 DB**: 캠페인(시즌, 기간, 히어로이미지, 활성화) + 추천상품(이미지, 가격, 쇼핑몰URL)
- **페이지**: `/seasons/spring-2026` 풀스크린 히어로 + CTA
- SNS 광고 랜딩 페이지로 활용, 개발자 없이 시즌마다 새로 제작

## 축 2: 업무 효율화 (다음 단계)

### Idea 14: 셀프 견적서 시스템 ---- 우선순위 최고 (수정)

**현재**: 엑셀 수동 → 메이크샵 견적서 기능도 있지만 커스터마이징 불가
**목표**: B2B 고객이 **직접** 상품 선택 + 수량 입력 → 견적서 자동 생성 (CS 감소)

2가지 접근:
1. **메이크샵 Open API 활용** — 상품 데이터(가격, 이미지, 재고)를 API로 가져와서 견적서 웹앱에서 활용
   - 메이크샵 API 제약: 시간당 조회 500회, 수정 권한 확인 필요
   - CORS 이슈 → Next.js API Route로 프록시 필요
2. **노션 DB 활용** — 별도 상품 DB를 노션에 관리 (메이크샵 API 한계 시)

웹사이트 화면:
- `/quotation` — B2B 고객이 직접 접속
- 상품 검색/카테고리 탐색 → 장바구니처럼 선택 → 수량/특이사항 입력
- → PDF 견적서 자동 생성 (회사 로고, 사업자 정보 삽입)
- → 이메일 발송 or 다운로드
- `/quotation/history` — (로그인 시) 내 견적 이력 조회

### Idea 15: 고객 매출 관리 (얼마에요 대체) ---- 우선순위 높음 (수정)

**현재**: 얼마에요 ERP에 수기 구매 매출만 입력, 월 5만원 비용
**목표**: 고객 매출내역 + 정보를 다른 시스템으로 이관 → 얼마에요 해지

접근 방법:
1. **메이크샵 Open API** — 주문/고객 데이터를 자동 수집
   - 주문 내역, 고객 정보, 매출 현황 자동 동기화
   - 수기 입력(오프라인 매출)만 별도 입력
2. **노션 DB로 관리** — 고객사, 상담이력, 매출이력 DB
3. **또는**: Google Sheets + Apps Script로 간단하게 (비용 0원)

**핵심**: 얼마에요의 "고객별 매출내역 조회" 기능만 대체하면 됨

### Idea 19: 업무 관리 시스템 (노션 기반) ---- 우선순위 높음 (확장)

**현재**: 업무 일정 관리 안됨, 재택근무 직원 관리 어려움
**목표**: 효율적인 팀 업무 관리 + 직원 온보딩 매뉴얼

노션만으로 구현 (웹 개발 불필요):
- **프로젝트 관리 DB**: 프로젝트명, 담당자, 마감일, 상태(할일/진행중/완료), 우선순위
- **업무 일지 DB**: 날짜, 담당자, 업무내용, 카테고리
- **업무 매뉴얼 페이지**: 부서별/직무별 SOP (표준 운영 절차)
  - 주문 처리 플로우
  - 고객 응대 가이드
  - 상품 등록 절차
  - SNS 콘텐츠 제작 가이드
- **주간 보고 템플릿**: 재택 직원 포함 전 직원 주간 보고

참고 템플릿: 노션 팀 워크스페이스 + 칸반 보드 + 캘린더 뷰

### Idea 18: 콘텐츠 캘린더 ---- 우선순위 중간

SNS 포스팅 계획을 노션에서 체계적으로 관리

노션만으로 구현 (웹 개발 불필요):
- **콘텐츠 DB**: 제목, 플랫폼(인스타/유튜브/블로그), 날짜, 상태(기획/촬영/편집/업로드), 캡션, 이미지링크
- 캘린더 뷰: 월간 포스팅 계획 한눈에
- 보드 뷰: 상태별 진행 현황 (기획 → 촬영 → 편집 → 완료)

### Idea 17: 고객 문의 자동화 (저렴한 대안) ---- 우선순위 중간 (수정)

**채널톡 대신 저렴한 대안**:

### Idea 20: 마케팅 분석 (메이크샵 API 활용) ---- 장기 (수정)

**메이크샵 Open API로 가능한 마케팅 분석**:

## 기술 스택

```
Framework:     Next.js 15 (App Router, TypeScript) — 별도 프로젝트
CMS 연동:      NotCMS (1차) + @notionhq/client (고급 시)
스타일링:       Tailwind CSS v4 + shadcn/ui 또는 커스텀
이미지:         next/image + API Route 프록시 (노션 이미지 만료 해결)
PDF 생성:      @react-pdf/renderer (견적서)
외부 API:      메이크샵 Open API (상품/주문 데이터)
배포:          Vercel (ISR, 무료 티어)
```

### 핵심 기술 이슈

1. **노션 이미지 URL 만료 (1시간)** → API Route 프록시로 해결
2. **메이크샵 API 제한** → 시간당 500회, CORS → Next.js API Route 프록시
3. **SEO** → Next.js SSG/ISR + generateMetadata + next-sitemap
